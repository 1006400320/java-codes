### git commit 规范  

> 原文 : [git-commit规范](https://momodiy.github.io/blog/git-commit-standard/)

####  提交 commit 的类型，包括以下几种

- feat: 新功能
- fix: 修复问题
- docs: 修改文档
- style: 修改代码格式，不影响代码逻辑
- refactor: 重构代码，理论上不影响现有功能
- perf: 提升性能
- test: 增加修改测试用例
- chore: 修改工具相关（包括但不限于文档、代码生成等）
- deps: 升级依赖

#### emoji规范
- :tada:	               初次提交
- :sparkles:	           引入新功能
- :bug:                    修复 bug
- :fire:	                   移除代码或文件
- :zap:	                   提升性能
- :pencil:	               撰写文档
- :lipstick:	           更新 UI 和样式文件
- :white_check_mark:	 增加测试
- :construction:	   工作进行中
- :wrench:	           修改配置文件
